# Screenshot preparation

Screenshots are intentionally not fabricated or committed as placeholders. Capture them manually after starting the API and frontend locally.

1. Start FastAPI and Next.js using the commands in the root README.
2. Use a 1440×900 viewport for `landing-page.png`, `analysis-input.png`, and `phishing-result.png`.
3. Use a 390×844 viewport for `mobile-analysis.png`.
4. On `/analyze`, click **Use safe example**, then **Analyze email** for the result capture.
5. Confirm the header says Connected or Degraded based on actual backend state; do not edit screenshots to imply availability.
6. Save files under `docs/images/` only when they are real captures, then add Markdown links to the root README.

Do not include real email, personal data, API secrets, or raw backend errors in screenshots.
